Setup > cd

cd

folder/reponame

Description

Use the cd command to navigate folders and files on your computer

Options

filename

You can type a folder or filename to navigate to it. You can also type multiple folders to navigate to a nested folder: folder1/nestedfolder/folder3

..

use cd .. to naviagte backwards. Like before, you can use multiple to navigate back up the folder tree: cd../../..

Setup > help

git help

--options

Description

With no options and no COMMAND or GUIDE given, the synopsis of the git command and a list of the most commonly used Git commands are printed on the standard output.

Options

--all

If the option --all or -a is given, all available commands are printed on the standard output.

--guide

If the option --guide or -g is given, a list of the useful Git guides is also printed on the standard output.

COMMAND --help

If an alias is given, git shows the definition of the alias on standard output. To get the manual page for the aliased command, use git COMMAND --help.

Projects > clone

git clone

--options

Description

Clones a repository into a newly created directory, creates remote-tracking branches for each branch in the cloned repository (visible using git branch -r), and creates and checks out an initial branch that is forked from the cloned repository’s currently active branch.

Options

--branch

Instead of pointing the newly created HEAD to the branch pointed to by the cloned repository’s HEAD, point to branch instead. In a non-bare repository, this is the branch that will be checked out. --branch can also take tags and detaches the HEAD at that commit in the resulting repository.

Working > add

git add

--options

Description

This command updates the index using the current content found in the working tree, to prepare the content staged for the next commit. It typically adds the current content of existing paths as a whole, but with some options it can also be used to add content with only part of the changes made to the working tree files applied, or remove paths that do not exist in the working tree anymore. The "index" holds a snapshot of the content of the working tree, and it is this snapshot that is taken as the contents of the next commit. Thus after making any changes to the working tree, and before running the commit command, you must use the add command to add any new or modified files to the index.

Options

filename.ext

Add the specified file or foldere to the commit or staging area

*

Adds all changed files and folders to staging area

Working > status

git status

--options

Description

Displays paths that have differences between the index file and the current HEAD commit, paths that have differences between the working tree and the index file, and paths in the working tree that are not tracked by Git (and are not ignored by gitignore[5]). The first are what you would commit by running git commit; the second and third are what you could commit by running git add before running git commit.

Options

--branch

Show the branch and tracking info even in short-format.

Working > commit

git commit

--options

Description

Create a new commit containing the current contents of the index and the given log message describing the changes. The new commit is a direct child of HEAD, usually the tip of the current branch, and the branch is updated to point to it (unless no branch is associated with the working tree, in which case HEAD is "detached" as described in git-checkout).

Options

--a

Tell the command to automatically stage files that have been modified and deleted, but new files you have not told Git about are not affected.

-m

Write a message in quotations to append to the commit
